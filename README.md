## 🗄️ Database Schema (PostgreSQL)

Tables created automatically on first run:

- `users` — accounts (username, email, password_hash, full_name)
- `url_checks` — URL scan history per user
- `statistics` — aggregate counts per user
- `saved_urls` — user's bookmarked URLs

---

## 🛡️ Detection Methods

The app combines two detection strategies:

1. **ML Model (CatBoost)** — trained on 12,000 URLs with 30 features
2. **Heuristic Analysis** — brand impersonation, suspicious TLDs, suspicious keywords, URL structure

If heuristic confidence ≥ 0.7, it overrides the ML verdict. Otherwise, both methods are combined.

## 🔐 Security Notes

- Passwords stored as Werkzeug hashes (never plaintext)
- Session-based auth with 1-hour lifetime
- CSRF-safe forms
- `SECRET_KEY` auto-generated by Render or set manually

## 📝 Migration from MySQL to PostgreSQL

The original `database.py` used `mysql-connector-python`. The deploy version uses `psycopg2-binary` with the same public API, so `app.py` and `auth.py` work unchanged. Key changes:

| Aspect | MySQL (original) | PostgreSQL (deploy) |
|--------|------------------|---------------------|
| `AUTO_INCREMENT` | `SERIAL` | |
| `TINYINT(1)` | `SMALLINT` | |
| `ON UPDATE CURRENT_TIMESTAMP` | Manual update in queries | |
| `%s` placeholders | `%s` (same) | |
| `mysql-connector-python` | `psycopg2-binary` | |
